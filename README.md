# 27.8Hz팀 프로젝트 Github 사용규칙
> 1. 항상 개발 시작전 pull 받고 개발 시작하기
> 2. 만약에 다른 사람이 수정하는 파일을 파일을 수정해야할 경우에는 그 사람에게 말하기
> 3. 각자 당일 수정할 파일 정하기
> 4. 자신의 하루 분량이 끝났을때만 push
> 5. push 하기전에 팀 톡방에 말하고 push하기, 말할때는 개발한 기능과 이름을 명시
